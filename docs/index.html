ğŸ“š How Bitcoin Addresses Work

1

Private Key Generation

A 256-bit random number is generated. This is your private key â€” a secret number that proves ownership.

e.g. e8f32e723decf4051aefac8e2c93c9c5b214313817cdb01a1494b917c8436b35
2

Public Key Derivation (ECDSA on secp256k1)

The private key is multiplied by the Generator Point G on the secp256k1 elliptic curve to produce a public key. This is a one-way operation â€” you cannot reverse it.

Public Key = Private Key Ã— G
3

Hashing (SHA-256 + RIPEMD-160)

The public key is first hashed with SHA-256, then the result is hashed again with RIPEMD-160, creating a 20-byte hash called the Public Key Hash (Hash160).

Hash160 = RIPEMD160(SHA256(public_key))
4

Address Encoding

The hash is then encoded using Base58Check (for legacy P2PKH) or Bech32/Bech32m (for SegWit/Taproot) to produce the final human-readable address.

P2PKH: 1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa
P2WPKH: bc1qw508d6qejxtdg4y5r3zarvary0c5xw7kv8f3t4
P2TR: bc1p0xlxvlhemja6c4dqv22uapctqupfhlxm9h8z3...
